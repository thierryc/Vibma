Vibma 包含 4 个内置 MCP 提示词，为 AI 代理提供最佳实践指导。代理在处理 Figma 设计时可以引用这些提示词获取上下文。

## design_strategy

使用 Figma 设计的最佳实践，涵盖：

- **先理解再创建** — 检查文档，发现现有样式和变量
- **使用设计令牌** — 优先使用 `fillStyleName`、`textStyleName` 和变量绑定，而非硬编码值
- **自动布局优先** — 对每个容器使用 `layoutMode`，避免绝对定位
- **命名规范** — 语义化名称，组件使用 `Property=Value` 模式
- **变量模式** — 将框架固定到特定模式（如 Dark）
- **质量检查** — 运行 `lint_node` 检测硬编码颜色、缺失的文本样式和布局问题

## read_design_strategy

阅读 Figma 设计的最佳实践。重点介绍使用 `get_selection()` 和基于选择的工作流来理解现有设计。

## text_replacement_strategy

在 Figma 设计中替换文本的系统方法：

1. **分析结构** — 扫描文本节点，识别逻辑分组（表格、列表、卡片）
2. **策略分块** — 按结构、空间区域、语义或组件划分工作
3. **渐进替换** — 克隆节点，逐块替换，每步导出并验证
4. **智能处理** — 适应容器约束，保持层级结构，适当缩放导出

## swap_overrides_instances

在组件实例之间转移内容覆盖的指南：

1. 识别源实例和目标实例
2. 使用 `instances(method: "get")` 提取覆盖
3. 将匹配的覆盖（文本、填充、样式）应用到目标子节点
4. 使用 `get_node_info` 和 `export_node_as_image` 进行验证
